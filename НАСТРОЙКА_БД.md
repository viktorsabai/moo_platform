# База данных: чтобы всё работало

Настройки из ЛК (меню, магазин, подписки) сохраняются в БД и применяются в приложении **только если база данных подключена**.

## Шаг 1: Создай `.env` в корне проекта

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/DATABASE?sslmode=require"
NEXTAUTH_SECRET="любая-случайная-строка-минимум-32-символа"
NEXTAUTH_URL="http://localhost:3000"
BOT_TOKEN=""
```

> **PostgreSQL:** можно использовать [Neon](https://neon.tech), [Supabase](https://supabase.com) или локальный Postgres.

## Шаг 2: Миграции и сиды

```bash
# Применить схему к БД
npx prisma migrate deploy

# Или для разработки (создаст миграцию если нужно)
npx prisma migrate dev --name init

# Заполнить демо-данными (меню, баннеры, подписки, настройки)
npx prisma db seed
```

## Шаг 3: Перезапуск

```bash
npm run dev
```

---

## Что делает seed

- Создаёт заведения `default` и `demo-2` с **включёнными** меню, магазином и подписками
- Добавляет категории, блюда, товары магазина
- Создаёт баннеры на главной
- Добавляет шаблоны подписок

## Проверка

1. Зайди в приложение
2. Войди (через Telegram или email)
3. Профиль → «Войти в кабинет» → выбери «Доставка на дом» → «Управление»
4. Настройки заведения → включи/выключи галочки → «Сохранить»
5. Открой главную / меню — изменения должны применяться сразу

Если при сохранении настройки появляется **«Ошибка базы данных»** — проверь `DATABASE_URL` в `.env`.
