## MVP рамки (актуально)

- Основной фокус: `меню ресторана + магазин + заказы + операционка продавца`.
- Подписки в клиентском продукте временно отключены:
  - пользователь видит заглушку;
  - CTA "Хочу подписку" создаёт лид;
  - лид уходит в Telegram команде заведения (OWNER/ADMIN/STAFF).
- Главная остаётся баннерной витриной, без искусственных fallback-обещаний.
- Доставка управляется продавцом через зоны и правила по зоне (стоимость/минимальный чек/окно).
- Для заказов используется единый pipeline статусов:
  `PENDING -> CONFIRMED -> PREPARING -> READY -> OUT_FOR_DELIVERY -> DELIVERED` (+ `CANCELLED`).
- Для исключения "обманутых ожиданий" пользователь всегда видит фактические ограничения доставки и текущий статус заказа.

## Основные функции

### 1. 🍽️ Каталог блюд (Menu)

**Описание**: Просмотр и выбор блюд из меню ресторана

**Возможности**:
- Просмотр блюд по категориям
- Поиск блюд по названию
- Фильтрация по категориям
- Сортировка по цене, популярности
- Детальная информация о блюде (описание, калории, аллергены)
- Добавление блюд в корзину

**Компоненты**:
- `DishCard` - карточка блюда
- `CategoryFilter` - фильтр категорий
- `SearchBar` - поисковая строка

**API**:
- `GET /api/dishes` - получение списка блюд
- `GET /api/dishes/:id` - получение блюда по ID
- `GET /api/categories` - получение категорий

---

### 2. 🛒 Корзина (Cart)

**Описание**: Управление товарами перед оформлением заказа

**Возможности**:
- Добавление/удаление блюд
- Изменение количества
- Просмотр итоговой суммы
- Применение промокодов (опционально)
- Сохранение корзины в localStorage
- Переход к оформлению заказа

**Компоненты**:
- `CartItem` - элемент корзины
- `CartSummary` - итоги корзины
- `CartDrawer` - выдвижная корзина
- `CartIcon` - иконка корзины с количеством

**Состояние**:
- Zustand store (`cart-store.ts`)

---

### 3. 📦 Заказы (Orders)

**Описание**: Оформление и отслеживание одноразовых заказов

**Возможности**:
- Оформление заказа из корзины
- Выбор адреса доставки
- Выбор времени доставки
- Оплата заказа
- Отслеживание статуса заказа
- Просмотр истории заказов
- Детальная информация о заказе

**Статусы заказа**:
1. `PENDING` - Ожидает подтверждения
2. `CONFIRMED` - Подтвержден
3. `PREPARING` - Готовится
4. `READY` - Готов
5. `OUT_FOR_DELIVERY` - В пути
6. `DELIVERED` - Доставлен
7. `CANCELLED` - Отменен

**Компоненты**:
- `OrderCard` - карточка заказа
- `OrderStatusBadge` - бейдж статуса
- `OrderTimeline` - временная линия заказа
- `OrderItem` - элемент заказа

**API**:
- `GET /api/orders` - получение заказов пользователя
- `GET /api/orders/:id` - получение заказа по ID
- `POST /api/orders` - создание заказа
- `PATCH /api/orders/:id` - обновление заказа (статус)

### 4. 🔄 Подписки (Subscriptions, post-MVP)

**Описание**: Регулярные доставки еды по подписке

**Возможности**:
- Создание подписки
- Выбор плана подписки (еженедельно, раз в 2 недели, ежемесячно)
- Выбор дней недели для доставки
- Выбор времени доставки
- Составление меню для подписки
- Управление подпиской (пауза, возобновление, отмена)
- Просмотр расписания доставок
- Автоматическое создание заказов по расписанию

**Планы подписки**:
- `WEEKLY` - Еженедельная доставка
- `BIWEEKLY` - Раз в две недели
- `MONTHLY` - Ежемесячная доставка

**Статусы подписки**:
- `ACTIVE` - Активна
- `PAUSED` - Приостановлена
- `CANCELLED` - Отменена
- `EXPIRED` - Истекла

**Компоненты**:
- `SubscriptionCard` - карточка подписки
- `SubscriptionForm` - форма создания/редактирования
- `DeliverySchedule` - расписание доставок
- `SubscriptionActions` - действия с подпиской

**API**:
- `GET /api/subscriptions` - получение подписок пользователя
- `GET /api/subscriptions/:id` - получение подписки по ID
- `POST /api/subscriptions` - создание подписки
- `PATCH /api/subscriptions/:id` - обновление подписки
- `POST /api/subscriptions/:id/pause` - приостановка подписки
- `DELETE /api/subscriptions/:id` - отмена подписки

**Сервисы**:
- `schedule-delivery.ts` - логика планирования доставок

## ✅ Эпик: Загрузка и выгрузка меню (закрыт)

**Реализовано:**

### Выгрузка
- Кнопка «Выгрузить» отправляет CSV меню владельцу в Telegram
- Формат: `category_slug`, `category_name`, `name`, `slug`, `price`, `description`, `emoji`, `tags`
- Шаблон для загрузки: `/api/admin/import/template` (скачать CSV)

### Загрузка
- Импорт CSV: категории, блюда, цены
- Поддержка формата: `category_slug`, `category_name`, `name`, `slug`, `price`, `description`, `emoji`, `tags`
- Упрощённый формат: `category`, `name`, `price` (slug генерируется автоматически)
- Парсер: BOM, точка с запятой, запятая
- Обратная связь: успех/ошибка под кнопками, toast

### UI и исправления
- Эмодзи категорий: редактирование в админке (клик по эмодзи → EmojiCarousel)
- Пустые категории показываются, эмодзи можно менять
- Подсказка «Меню и товары»: «36 блюд · 4 в маг.» вместо «4 поз.»
- Кнопки «Управлять» и «Подробнее» — на всю ширину, по центру

### Деплой
- `next.config.js`: `serverComponentsExternalPackages` для form-data/axios
- `typescript.ignoreBuildErrors: true` — обход зависания сборки на Vercel

## User Flow

### Оформление одноразового заказа

1. Пользователь просматривает меню
2. Добавляет блюда в корзину
3. Переходит в корзину
4. Выбирает адрес доставки
5. Выбирает время доставки
6. Переходит к оплате
7. Оплачивает заказ
8. Получает подтверждение
9. Отслеживает статус заказа

### Оформление подписки

1. Пользователь выбирает "Подписка"
2. Выбирает план подписки (еженедельно/раз в 2 недели/ежемесячно)
3. Выбирает дни недели для доставки
4. Выбирает время доставки
5. Составляет меню для подписки
6. Выбирает адрес доставки
7. Оплачивает подписку
8. Получает подтверждение
9. Система автоматически создает заказы по расписанию

## Технические детали

### Состояние заказа

Заказы проходят через следующие этапы:
```
PENDING → CONFIRMED → PREPARING → READY → OUT_FOR_DELIVERY → DELIVERED
                                                              ↓
                                                          CANCELLED (может быть на любом этапе)
```

### Подписки и доставки

- При создании подписки автоматически создаются записи `SubscriptionDelivery` на основе выбранных дней
- По наступлению даты доставки система может автоматически создать заказ (через cron job или очередь задач)
- Статус доставки обновляется по мере выполнения заказа

### Безопасность

- Все API routes защищены проверкой сессии
- Платежи обрабатываются через Stripe (PCI DSS compliant)
- Пароли хранятся в хешированном виде (bcrypt)
- JWT токены для сессий
