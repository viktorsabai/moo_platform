# Схема БД и Prisma Migrate

## P3005: «The database schema is not empty» при `migrate deploy`

Типично, если база уже заполнена (`db push`, ручные SQL, старый проект), а таблицы `_prisma_migrations` ещё не ведут историю так, как ждёт **первый** полный `deploy`.

**Что сделать:**

1. Убедиться, что у **каждой** папки в `prisma/migrations/` есть файл `migration.sql` (у вас папка `20250204000000_add_category_prep_max` была пустой — без этого `migrate resolve` даёт P3017).
2. Пометить уже «фактически применённые» миграции без повторного SQL (baseline), **по порядку имён папок**:
   ```bash
   npx prisma migrate resolve --applied "20250204000000_add_category_prep_max"
   npx prisma migrate resolve --applied "20250207000000_add_min_max_dishes_per_delivery"
   # … остальные по списку из `ls prisma/migrations`
   ```
   Либо один раз пометить все, если схема точно совпадает с тем, что должны были сделать миграции (как после аккуратного `db push`).
3. Затем: `npx prisma migrate deploy`.

Если после `deploy` миграция **упала** (например, `relation already exists`), а объекты в БД уже есть — это **P3018 / P3009**:

```bash
npx prisma migrate resolve --applied "ИМЯ_ПАПКИ_УПАВШЕЙ_МИГРАЦИИ"
npx prisma migrate deploy
```

**Альтернатива только для локалки:** без истории миграций — `npx prisma db push` (как ниже для `menuEnabled`).

## Ошибка: "The column `menuEnabled` does not exist"

Если видите эту ошибку — схема БД не синхронизирована с Prisma.

**Локальная БД:**
```bash
npx prisma db push
```

**Production (Vercel):** если приложение на Vercel, нужен push в production-базу:
```bash
npm run db:push:prod
```
(использует `.env.vercel.production.local` — сначала `vercel env pull` если нужно)

Это добавит недостающие колонки в таблицу `AppSettings` (menuEnabled, storeEnabled, subscriptionEnabled).

После этого перезапустите/передеплойте приложение.
